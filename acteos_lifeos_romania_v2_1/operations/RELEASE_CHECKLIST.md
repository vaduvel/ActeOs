# Release checklist

- [ ] scope și release notes aprobate;
- [ ] OpenAPI compatibility și clienți generați;
- [ ] migrations testate pe copie staging și plan de rollback;
- [ ] unit, integration, contract, E2E, a11y și security gates verzi;
- [ ] niciun secret sau PII în artefacte/loguri;
- [ ] bundle-ul de reguli are manifest, hash, signature metadata și four-eyes approval;
- [ ] critical source freshness este verde;
- [ ] feature flags și kill switches verificate;
- [ ] observability dashboards și alerte active;
- [ ] staged rollout definit;
- [ ] owner on-call și rollback decision clar;
- [ ] monitorizare post-release și verdict în jurnal.
